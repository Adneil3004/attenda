# Session Changelog: Architecture Consolidation

This document summarizes the major changes made during this session to stabilize the Attenda project, moving from a client-side heavy logic to a secure, backend-driven architecture.

## 🔐 Security & Authentication
- **OIDC Discovery**: Configured .NET JWT Bearer middleware to use OIDC Discovery (via `options.Authority`). This automatically handles **RS256** key rotation from Supabase.
- **Identity Mapping**: Implemented `ICurrentUserService` and `CurrentUserService` to bridge Supabase `sub` claims with the standard .NET `ClaimTypes.NameIdentifier`.
- **Debug Verbosity**: Enabled `IdentityModelEventSource.ShowPII` for easier troubleshooting of JWT validation issues in development.

## 🛣️ Backend API & Routing
- **Route Segregation**: Fixed multiple **404 Not Found** errors by providing more specific routes in `EventsController` and `GuestsController`.
- **Dashboard Consolidation**: Created a unified dashboard endpoint (`api/Events/dashboard`) that combines event metadata and guest statistics in a single call.
- **Guests CRUD**: Implemented full CRUD logic for guests (Update, Batch Delete, Import) using MediatR and CQRS patterns.

## 💻 Frontend Evolution
- **Client Reliability**: Switched API calls from `localhost` to `127.0.0.1` to avoid IPv6/IPv4 resolution conflicts in certain environments.
- **Logic Migration**: Refactored `Guests.jsx` and `Overview.jsx` to stop using direct Supabase SDK calls for business logic. They now interact with the .NET Backend via the `apiClient`.
- **Enhanced CSV Import**: Improved CSV parsing logic with better column mapping and support for multiple dietary restrictions (semicolon separated).

## 📁 Infrastructure & Domain
- **Identity Abstraction**: Added infrastructure services to handle user context without leaking JWT details into the application layer.
- **Domain Updates**: Updated `Guest` and `Event` aggregates to support the new feature set (Guest Groups, Capacity Tiering, etc.).
